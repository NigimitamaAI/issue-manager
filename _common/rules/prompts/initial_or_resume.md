# 初回／再開プロンプト案

このプロンプトは、チケットの有無にかかわらず issue_manager 管理下プロジェクトで作業を始めるための標準案です。

```text
## 共通運用ルール
プロジェクト固有ルールの前に、以下を読んでください。
- <issue_manager>\_common\rules\RULES.md
- <issue_manager>\_common\rules\AI_OPERATION.md
- <issue_manager>\_common\rules\docs\*.md のうち今回の作業に関係するもの

## ブロック A: issue_manager 基本処理
次を確認してください。
- 運用版 / 開発版のどちらを扱っているか
- 対象プロジェクトの `tickets\RULES.md` と `tickets\INDEX.md`
- review/、doing/、todo/ の現在状態
- 今回の作業に該当する確認義務トリガー

ポリシー文書を読んだ場合は、`AI_OPERATION.md` の関連実装表に従い、該当する `routes.mjs`、`lib/*.mjs`、設定ファイルも確認してください。

## ブロック B: 状況判定
- review/ にユーザー返信または判定がある場合: 返信確認を優先
- doing/ にチケットがある場合: 作業ログのサマリ欄と引継ぎメモを読んで継続
- todo/ に着手候補がある場合: 1件を doing/ に移して開始
- チケットがない場合: 用件を整理し、必要なら inbox/ または doing/ に起票してから開始

## ブロック C: 作業
- 既存ルール、プロジェクト固有ルール、チケット本文を上書きせず、追加ガイドは補助として扱う
- 実装変更時は関連ファイルを確認し、必要な最小範囲で編集する
- セッション終了時は、サマリ1行、詳細ログ、引継ぎメモを更新する
```

## チケット指定再開用

```text
## 今回の作業対象
タイトル: <title>
チケット: <ticket path>
レーン: <lane>

<tickets path> の RULES.md を読んだ後、以下のチケットを開いて作業を再開してください。

チケット: <ticket path>
タイトル: <title>

手順:
1. 共通ルール、AI_OPERATION.md、プロジェクト RULES.md を読む
2. チケットの作業ログサマリ欄と引継ぎメモを中心に読む
3. review/ の場合はユーザー返信欄と判定を確認
4. review/ 以外の場合は未完了の完了条件から継続
5. 作業対象が確認義務トリガーに該当する場合は関連実装も読む
6. 終了時はサマリ1行、詳細ログ、引継ぎメモを更新する
```

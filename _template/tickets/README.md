# <プロジェクト名> — チケット運用

このディレクトリは issue_manager で管理されるチケット領域です。

## Claudeへの初回指示（コピペ用）

```
<プロジェクトのフルパス>/tickets/RULES.md と INDEX.md を読んで、
todo/ から次に着手するチケットを選んで作業を始めてください。
```

例（環境変数 `ISSUE_MANAGER_ROOT` を `G:\claudedir` に設定している場合）:
```
G:\claudedir\<プロジェクト名>\tickets\RULES.md と INDEX.md を読んで、
todo/ から次に着手するチケットを選んで作業を始めてください。
```

## セッション再開時の指示（コピペ用）

```
<プロジェクトのフルパス>/tickets/ の doing/ にあるチケットから作業を再開してください。
なければ INDEX.md を確認して次のチケットを選んでください。
```

## フォルダの意味

| フォルダ | 意味 | 遷移トリガー |
|----------|------|--------------|
| todo/    | 未着手 | 新規作成時 |
| doing/   | 着手中 | セッション開始時に1つ選んで移動 |
| blocked/ | ブロック中 | 外部待ち・判断待ち発生時 |
| done/    | 完了済み | 完了条件を全て満たしたとき |
| .trash/  | ソフト削除退避先 | 画面から削除時 |

## 設計思想

- **トークン節約**: 毎回プロジェクト全体を読まず、アクティブな1チケット＋INDEXだけで立ち上がる
- **引継ぎ容易**: セッション跨ぎ・別Claudeでの継続でもコンテキストが失われない
- **監査可能**: 詳細ログを原文のまま残し、判断経緯を後から追える
- **2層ログ**: サマリ欄で軽量運用、詳細ログ欄で完全性を担保

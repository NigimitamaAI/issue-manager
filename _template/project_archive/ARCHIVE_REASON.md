# プロジェクト・アーカイブ理由書

> **使い方**: プロジェクト全体を `G:\_archive\projects\` 配下に移動して凍結する際、移動先ルートに本ファイル（実値で埋めたもの）を必ず添える。これは「なぜこのプロジェクトを動的開発から外したか」「再評価の条件」「元位置からの参照経路」を将来の自分・他AI に説明する記録。
> テンプレ自体（issue_manager 側の本ファイル）は雛形なので編集しない。プロジェクト個別の理由書は移動先ディレクトリで `ARCHIVE_REASON.md` として作成する。

## メタ情報

| 項目 | 値 |
| :--- | :--- |
| アーカイブ日 | YYYY-MM-DD |
| 元パス | `G:\path\to\original\project\` |
| アーカイブ先 | `G:\_archive\projects\<project_name>_YYYYMMDD\` |
| 元プロジェクト名 | `<project_name>` |
| 関連リポジトリ | git 化されていれば URL、なければ「git 未設定」 |
| 起票元チケット | `<起票したプロジェクト>/tickets/<lane>/<filename>` |
| 起票元 AI | `claude-main` / `Claude.ai web` / `Codex` 等 |
| 判断者 | `<ユーザー名>` |

## アーカイブ種別（該当項目に [x]）

- [ ] **段階廃棄** — 将来的な完全削除を前提に、暫定凍結
- [ ] **参考凍結** — 削除予定はないが、現役からは外す（後継版との並行維持は終了）
- [ ] **一時退避** — 数ヶ月以内に判断見直し予定
- [ ] **その他** — 自由記述で説明
